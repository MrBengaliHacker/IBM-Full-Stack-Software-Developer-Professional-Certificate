# 📝 Graded Quiz – The Software Development Lifecycle

### 1. Your company is looking to hire a new software engineer. During the interview process, you ask candidates to define software engineering. Which of the following is the most accurate description?

- **A.** The application of structured principles to arrange the components of a software system

- **B.** Any task involved in software development

- **C.** The use of scientific methods to design and build software

- **D.** The complete collection of activities involved in software development

✅ **Correct Answer:** **C. The use of scientific methods to design and build software**

---

### 2. A healthcare company is developing a new patient management system. To ensure smooth development and minimize risks, they choose to follow the Software Development Life Cycle (SDLC). Which of the following best describes a key advantage for a business to follow the SDLC?

- **A.** Focuses exclusively on defining the roles of project stakeholders.

- **B.** Provides developers with complete autonomy over the project timeline.

- **C.** Helps in writing user manuals and technical documentation.

- **D.** Ensures a structured approach for improving efficiency and reducing risks.

✅ **Correct Answer:** **D. Ensures a structured approach for improving efficiency and reducing risks.**

---

### 3. After a software system is deployed, ongoing updates and issue resolution are required to ensure smooth operation. Which task is typically performed during the maintenance phase of the SDLC?

- **A.** Performing initial system integration testing.

- **B.** Developing core system functionalities.

- **C.** Writing the initial system requirements.

- **D.** Resolving minor software bugs and performance issues.

✅ **Correct Answer:** **D. Resolving minor software bugs and performance issues.**

---

### 4. During the final testing of a new e-commerce system, the team wants to confirm that the software performs as defined in the specifications. Which of the following activities should they perform in the testing phase in this scenario?

- **A.** Define system components and behaviors.

- **B.** Checking software against requirements.

- **C.** Create support guides for users.

- **D.** Launch internal alpha for testing.

✅ **Correct Answer:** **B. Checking software against requirements.**

---

### 5. You want to incorporate multiple subsystems into your new inventory management platform. The project manager wants a document that contains user and software requirements to focus on the entire scope of the system. Which document should you use in this case?

- **A.** System Requirements Specification (SysRS)

- **B.** Software Requirements Specification (SRS)

- **C.** User Requirements Specification (URS)

- **D.** Interface Control Document (ICD)

✅ **Correct Answer:** **A. System Requirements Specification (SysRS)**

---

### 6. An organization is planning its software development approach with a strict step-by-step development process in which one phase must be completed before starting a new phase. Which Software Development Life Cycle (SDLC) model does this scenario describe?

- **A.** Verification

- **B.** V-shape

- **C.** Agile

- **D.** Waterfall

✅ **Correct Answer:** **D. Waterfall**

---

### 7. What is the primary objective of software validation that focuses on user experience?

- **A.** Monitoring infrastructure performance metrics.

- **B.** Making sure the application is usable and accessible.

- **C.** Tracking system modification consequences.

- **D.** Measuring system resource consumption.

✅ **Correct Answer:** **B. Making sure the application is usable and accessible.**

---

### 8. Your team has completed developing and testing new software. Which process will you use to create a detailed technical document that helps future developers understand the system architecture and operation?

- **A.** Code quality

- **B.** Software design

- **C.** Software testing

- **D.** System documentation

✅ **Correct Answer:** **D. System documentation**

---

### 9. A product development team is discussing prioritizing features for their next release based on customer requirements. Which of the following roles makes decisions for feature priority?

- **A.** Software architect

- **B.** UX designer

- **C.** Stakeholder

- **D.** Product Owner

✅ **Correct Answer:** **D. Product Owner**

---

### 10. In the early planning stages of a complex enterprise software project, the team reviews documentation requirements. Which type of documentation details the software system's expected features and functionality?

- **A.** Requirements documentation

- **B.** Technical documentation

- **C.** User documentation

- **D.** Process documentation

✅ **Correct Answer:** **A. Requirements documentation**
