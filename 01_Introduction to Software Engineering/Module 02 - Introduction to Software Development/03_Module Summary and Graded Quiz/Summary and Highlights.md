## 1) Web and Cloud Development

- Websites use **client-server communication** to exchange data.

- **HTML** provides structure, **CSS** adds styling, and **JavaScript** adds interactivity.

- Websites can contain **static** and **dynamic** content.

- Cloud applications use cloud infrastructure to provide scalable and reliable services.

## 2) Front-end and Back-end Development

- **Front-end development** focuses on the user interface using HTML, CSS, and JavaScript.

- **Back-end development** handles business logic, databases, APIs, authentication, and security.

- **Full-stack developers** work on both front-end and back-end development.

## 3) Teamwork and Agile Practices

- Software engineering teams collaborate through planning meetings, code reviews, walkthroughs, and retrospectives.

- **Squads** are small Agile teams that typically include a squad leader and software engineers.

- **Pair programming** improves code quality, knowledge sharing, and problem-solving.

## 4) Application Development Tools

### Version Control
- Git tracks changes to source code.
- GitHub hosts and manages Git repositories.

### Libraries
- Provide reusable code for specific features.
- Developers decide when to use library functions.

### Frameworks
- Provide a standard structure and workflow for application development.
- Use **Inversion of Control (IoC)** to manage program flow.

## 5) Build and Deployment Tools

- **CI/CD** automates integration, testing, and deployment.

- **Build tools** compile, package, test, and deploy applications.

- **Packages** bundle application files and dependencies.

- **Package managers** install, update, and manage software packages.

## 6) Software Stacks

- A **software stack** combines technologies used to build applications.

- A **technology stack** also includes hardware and infrastructure.

- Common software stacks include:
  - **LAMP**
  - **MEAN**
  - **MERN**
  - **MEVN**

- **MEAN** uses JavaScript throughout the stack.

- **LAMP** is a mature open-source stack that works well with relational databases.
