# 📝 Graded Quiz – Software Architecture, Design, and Deployment Topologies

### 1. Your team is tasked with creating a scalable enterprise application, considering that the earlier design choices will have a long-lasting impact on the application's structure and future development process. Which of the following aspects should a team focus on?
- A. It represents the earliest design decisions.
- B. It describes how the application should be coded.
- C. It captures implementation details.
- D. The choice of technology stack drives the software architecture.

✅ **Correct Answer:** **A. It represents the earliest design decisions.**

---

### 2. Which of the following would typically be created as a result of software architecture design?
- A. A collection of system test cases
- B. Unified Modeling Language (UML) diagrams
- C. The administrator's user guide
- D. A completed Software Requirements Specification (SRS)

✅ **Correct Answer:** **B. Unified Modeling Language (UML) diagrams**

---

### 3. You are tasked to analyze the modelling behavior of the smart thermostat system for its transition between heating, cooling, and idle states. Which of the following best describes a state transition diagram in Unified Modeling Language (UML)?
- A. A diagram showing various states in the system and events that cause transitions between those states.
- B. A diagram that shows structural relationships between software modules.
- C. A diagram that models the dynamic interaction between the system's objects.
- D. A diagram that represents step-wise operation systems.

✅ **Correct Answer:** **A. A diagram showing various states in the system and events that cause transitions between those states.**

---

### 4. A new software team was provided with the Unified Modeling Language (UML) diagram to understand the system's components and their interactions in software architecture. What is the benefit of UML diagrams in software architecture?
- A. They define the logic for each method call.
- B. They provide a visual overview for onboarding and collaboration.
- C. They document required input parameters for object creation.
- D. They ensure every class has JavaScript compatibility.

✅ **Correct Answer:** **B. They provide a visual overview for onboarding and collaboration.**

---

### 5. A startup company wants to design an application in which each node captures requests and provides services to the other nodes without a central server. Which of the following architectures is best suited in this scenario?
- A. Three-tier architecture
- B. Peer-to-peer architecture
- C. Client-server architecture
- D. Microservices architecture

✅ **Correct Answer:** **B. Peer-to-peer architecture**

---

### 6. A software development team wants to represent a hospital's patient management system using object-oriented concepts. How will they define the relationship between patient Naya Patel and the patient class in the object-oriented system?
- A. Naya Patel is an instance of the patient class with specific attribute values.
- B. The patient class inherits its properties from Naya Patel's specific attributes.
- C. Naya Patel is a blueprint to create various patient classes.
- D. Naya Patel is an abstract representation of the patient class.

✅ **Correct Answer:** **A. Naya Patel is an instance of the patient class with specific attribute values.**

---

### 7. A developer builds a system where the user interface directly communicates with a server that handles data processing. No separate application logic layer is used. What type of architecture is this?
- A. An arrangement where all nodes act as both client and server
- B. A setup where two tiers refer to logic and database
- C. An event-driven pattern involving producers and consumers
- D. A 2-tier architecture with client interfaces talking directly to a server

✅ **Correct Answer:** **D. A 2-tier architecture with client interfaces talking directly to a server**

---

### 8. You are designing a new social media site and want to build it using modular services that interact via APIs. Which architecture pattern is appropriate for this scenario?
- A. Microservices architecture
- B. Two-tier architecture
- C. Peer-to-peer architecture
- D. Event-driven architecture

✅ **Correct Answer:** **A. Microservices architecture**

---

### 9. While launching an application for public users, it is important to secure their information. Which environment should an organization use to launch this application?
- A. QA teams use a verification system to test application components.
- B. The coding team uses the development environment to build software.
- C. The staging setup imitates the live environment without public access.
- D. The production team deploys a fully operational production system for general users.

✅ **Correct Answer:** **D. The production team deploys a fully operational production system for general users.**

---

### 10. A company wants to ensure that external threats, such as malware or hackers, cannot directly access the internal network in the production environment. Which component is most suitable for the requirement?
- A. Application server
- B. Proxy server
- C. Load balancer
- D. Firewall

✅ **Correct Answer:** **D. Firewall**
